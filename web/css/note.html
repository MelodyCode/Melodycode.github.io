# 盒子模型

又名：盒模型，框模型

  • css将页面中所有的元素都设置成一个矩形的盒子；设置成盒子之后，页面布局就变成将不同的盒子放到不同的位置
  • 每一个盒子都由以下几部分组成：
    • 内容区（content）：
    • 内边距：(padding)：内容区和边框之间的距离
    • 边框（border）： ps 以上三者均会影响盒子大小
    • 外边距：(margin)不会影响盒子可见框的大小，会影响盒子实际空间大小，指定当前盒子与其他盒子的距离
  • 一个盒子可见框的大小、由盒子的内容区，内边距，边框共同决定； 计算盒子大小时，将三者加起来
  • 元素在页面中是按照自左向右顺序排列的，所以默认情况下设置，左上边距会移动元素自身，设置右下边距，会移动其他元素。
/* html
<div class="box1"></div>
*/

/*css*/
.box{
/*内容区：元素中所有子元素和文本内容在内容区中排列，内容区大小有width和height两个属性设置*/
	width:200px;
	height:200px;
	background-color:#bfa;

/*边框：属于盒子边缘，边框里面属于盒子内部，出了边框就是盒子外部，要设置边框，至少设置三个样式：(缺一不可)
边框大小也会影响整个盒子的大小
	边框宽度：border-width：10px;--->上下左右都是10px；
			  border-width:10px 20px 30px 40px;
			  四个值顺序：上，右，下，左
			  三个值：上 左右 下
			  两个值：上下 左右
			  border-xxx-width:
			  xxx可以是：top,left,right,bottom;
			  用来单独制定某个边框
	边框颜色：border-color
		规则和width一样;
		可以省略不写，若省略了就自动使用color（一般是前景色）的值
	边框样式：border-style：
	    规则和width一样；默认值为none，表示没有边框
		solid：实线；
		dotted：点状虚线
		dashed：虚线
		double：双线
*/
/*简写属性（常用）：border：10px solid red;
		单独设置border-xxx:
		xxx为top，left,right,bottom
*/
	border-width:10px;
    border-color:red;
	border-style:solid;

/*内边距(padding):内容区和边框之间的距离
一共有四个方向：
	padding-top:
	padding-bottom:
	padding-left:
	padding-right:
	简写：padding：上 右 下 左
特点：内边距的设置会影响盒子大小；
背景颜色会延伸到内边距;
一个盒子可见框的大小、由盒子的内容区，内边距，边框共同决定；
计算盒子大小时，将三者加起来
*/

/*外边距：
	margin-top:上边距，设置正值，元素会往下移动 
	margin-bottom:下边距，设置正值，他下面的元素会往下移动
	margin-left:左边距，设置正值，元素会往又移动 
	margin-right:默认情况下，不会产生任何效果
	简写：margin：上 右 下 左；
负值，则往相反方向移动
	
*/
}

  • 盒子的水平布局：
    • 元素在其父元素中，水平方向的位置由以下几个属性共同决定：
      • margin-left:
      • border-left:
      • padding-left:
      • width://默认值就是auto
      • padding-right:
      • border-right:
      • margin-right:
    • 上述七个值之和==父元素内容区的宽度（必须满足），如果相加结果使等式不成立，则称为过渡约束，则等式会自动调整
      • 若上述七个之中没有auto情况，浏览器会自动调整margin-right使等式满足
      • 上述七个值中有三个值可以设置为auto：若有属性设置为auto，则浏览器自动调整auto那个值使等式成立
        • width：
        • margin-left:
        • margin-right:
      • 如果将一个宽度和一个外间距设置为auto，则宽度会调整到最大，设置为auto的外间距自动为0
      • 如果将三个值都设置为auto，则外边距都是0，宽度最大
      • 如果两个外边距设置为auto，宽度固定值，则外边距会被设置为相同的值--所以利用这一点使一个元素在其父元素中居中
/*使一个元素在其父元素中居中常用方法*/
    width:xxxxpx;
	margin:0 auto;

  • 盒子垂直方向的布局：

    • 默认情况下父元素（父元素不设定高度的情况下）的高度被内容撑开
    • 若子元素是在父元素的内容区排列的
      • 如果子元素的大小超过了父元素，则子元素会从父元素中溢出
      • 使用overflow属性来设置父元素如何处理溢出的子元素（overflow-x,overflow-y）
      • 可选值：

        • visible：默认值，子元素会从父元素中溢出，在父元素外部显示；

        • hidden：超出内容会被隐藏，不会被显示

        • scroll：生成垂直/水平方向滚动条

        • auto;根据需要生成滚动条

  • 相邻的垂直方向外边距的折叠（重叠）：

    • 相邻垂直方向外边距会发生重叠现象
    • 兄弟元素：
      • 兄弟元素间的相邻垂直外间距会取两者之间值较大的（两者都是正值）
      • 特殊情况：如果相邻的外间距一正一负，取两者的和；若均为负值，则取两者中绝对值较大的
      • 兄弟元素外间距的重叠对开发有利，不需要处理
    • 父子元素：
      • 父子元素间的相邻外间距，子元素会传递给父元素
      • 父子外间距的折叠会影响页面整体布局，必须处理;
        • 待处理。。。。

  • 行内元素的盒模型：

    • 行内元素不支持设置width和height

    • 行内元素可以设置padding，但是垂直方向不会影响页面的布局

    • 行内元素可以设置border，但是垂直方向不会影响页面的布局

    • 行内元素可以设置margin，但是垂直方向不会影响页面的布局

    • display用来设置元素显示的类型

      • 可选值：
        • inline:将元素设置为行内元素
        • block：将元素设置为块元素
        • inline-block：将元素设置为行内块元素，特点：既可以设置为宽高，又不独占一行；（少用）
        • table：将元素设置为表格
        • none：元素不在页面显示
      • visibility：用来设置元素的显示状态：
        • 可选值：
          • visible：默认值
          • hidden：元素在页面中隐藏不显示，但是占着位置

  • 盒子的大小：
    • 默认情况下：盒子可见框的大小：内容框、内边距、和边框共同决定
    • box-sizing：用来设置盒子大小的计算方式（设置width和height的作用）

# 浮动（float）

  • 通过浮动可以使一个元素向其父元素的左侧和右侧浮动，使用float属性来设置元素的浮动-----目的让元素横向布局
  • 可选值：
    • none：默认值，元素不浮动；
    • left:元素向左
    • right：元素向右
  • 元素设置浮动后，水平布局的等式不需要强制成立（可以没有外边距）；
  • 元素会完全从文档流中脱离，不在占用文档流位置，所以元素下面还在文档流中的元素会自动向上移动
  • 浮动的特点：
    • 浮动元素会完全脱离文档流，不在占据文档流的位置
    • 设置浮动以后，元素会向父元素的左侧或右侧移动
    • 浮动元素默认不会从父元素中移出
    • 浮动元素向左或向右移动时不会盖住其他的浮动元素
    • 如果浮动元素上面是没有浮动的块元素，则浮动元素没办法上移
    • 浮动元素不会超过上面浮动的兄弟元素，最多和他们一样高，（但可以通过html结构改变---交换兄弟）
    • 浮动元素不会盖住文字，文字会自动环绕在浮动元素周围-----可以利用该特性设置文字环绕图片效果
    • 元素脱离文档流的特点：元素设置浮动后从文档流中脱离，脱离后元素的一些特点会发生变化：
      • 块元素：
        • 块元素不会独占一行
        • 脱离文档流以后，元素宽度和高度会默认被内容撑开
      • 行内元素：
        • 行内元素脱离文档流后变成块元素，特点和块元素一样
      • 脱离文档流以后，不再区分块和行内了

  • 高度塌陷：

    • 在浮动布局中，父元素的高度是默认被子元素撑开的；
      • 当子元素浮动时，其会完全脱离文档流，子元素从文档流中脱离，将会无法撑起父元素的高度，导致父元素的高度丢失
      • 父元素高度丢失以后，其下面的元素会自动上移，导致页面布局混乱------问题严重，必须解决

  • 如何解决高度塌陷：BFC

    • BFC：Block Formatting Context,块级格式化环境
    • BFC是CSS中一个隐含的属性，可以为一个元素开启BFC，开启BFC后元素会变成一个独立的布局区域
    • 元素开启BFC的特点：
      • 开启BFC元素不会被浮动元素覆盖
      • 开启BFC的元素，子元素和父元素外边距不会重叠
      • 开启BFC的元素，可以包含浮动的子元素
    • 如何开启BFC：
      • 设置元素的浮动（不推荐）
      • 将元素设置为行内块元素（不推荐）
      • 将元素的overflow设置为非visible的值
        • 常用的方式：将元素设置overflow：hidden，开启BFC,使其可以包含浮动元素

  • clear:

    • 清除浮动元素对当前元素产生的影响
    • 可选值：
      • left：清除左侧浮动元素对当前元素的影响
      • right：清除右侧浮动元素对当前元素的影响
      • both：清除两侧浮动元素中最大的影响
    • 原理：设置清除浮动以后，浏览器会自动为元素加一个上外边距，一遍其位置不受其他元素的影响

  • 高度塌陷最终解决方案：没有副作用

    • 伪元素: ::after----->默认是行内元素

  • 如何解决高度重叠问题？

# 定位（Position）

  • 定位：

    • 定位是一种更加高级的布局手段，可以将元素摆放到页面任意位置
    • 使用position属性来设置定位：
      • 可选值：
        • static：默认值，元素是静止的没有开启定位
        • relative：相对定位
        • absolute：绝对定位
        • fixed：固定定位
        • sticky：粘滞定位

  • 相对定位：

    • position:relative
    • 特点：
      • 元素开启相对定位以后，如果不设置偏移量元素不会发生任何变化
      • 相对定位是参照元素在文档流的位置定位的
      • 相对定位会提升当前元素的层级
      • 相对定位不会使元素脱离文档流，不会改变元素的性质
    • 偏移量（offset）：---->只在开启定位后生效
      • top：定位元素和定位位置上边的距离：
      • bottom：定位元素和定位位置下边的距离
        • 垂直方向的定位由top和bottom两个属性控制，一般只需要用其中之一；top越大元素越往下移，bottom越大元素越往上移动
      • left：定位元素和定位位置左边的距离
      • right：定位元素和定位位置右边的距离
        • 水平方向的定位由left和right两个属性控制，left越大，元素越靠右，right越大，元素越靠左

  • 绝对定位：

    • 特点：
      • 元素开启绝对定位以后，如果不设置偏移量元素的位置不会发生任何变化
      • 元素开启绝对定位以后，元素从文档流中脱离
      • 绝对定位会改变元素的性质，行内元素会变成块，块内的宽高被内容撑开
      • 绝对定位使元素提升一个层级
      • 绝对定位元素相对于其包含块定位的
        • 包含块（containing block）： 
          • 正常情况下，包含块就是离当前元素块最近的祖先元素
          • 开启绝对定位的情况下，包含块是离他最近的开启定位（position的值不是static）的祖先元素；如果所有的祖先元素都没有开启定位，则相对于根元素定位
          • html：根元素，又名：初始包含块

  • 固定定位：

    • position：fixed
    • 特点：
      • 也是一种绝对定位，大部分特点和绝对定位一样
      • 不同点：固定定位永远参照于浏览器的视口进行定位；不会随网页滚动条滚动

  • 粘滞定位:

    • position:sticky

    • 特点：和相对定位的特点基本一致，不同的是粘滞定位到达某一位置后将元素固定

    • 缺点：兼容性差
