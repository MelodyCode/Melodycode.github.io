# Vue高级特性

  • 自定义v-model

    • 双向绑定：都是通过数据的改变去驱动 DOM 视图的变化，而双向绑定除了数据驱动 DOM 外， DOM 的变化反过来影响数据，是一个双向关系，vue中用v-model实现
    • 注意：
      • 上面的input使用了:value 而不是 v-model
      • 上面的change和model.event 要对应起来
      • text属性对应起来

  • $nextTick

    • Vue是异步渲染
    • data改变后，DOM不会立刻渲染
    • $nextTick会在DOM渲染完再回调，以获取最新DOM节点
    • 页面渲染时会将data的修改做整合，多次data修改只会渲染一次

  • slot

    • 基本使用 ：父组件往子组件插入内容

    • 作用域插槽：子组件有作用域data，如何让父组件获取到？ 父 插槽

    • 具名插槽：多个插槽一一对应

  • vue动态组件

    • :is = "component-name"用法
    • 需要根据数据，动态渲染的场景，即组件类型不确定。eg：加载新闻 根据type渲染对应类型的组件

  • 异步组件：大组件什么时候用到什么时候异步加载

    • import()函数
    • 按需加载

  • vue如何缓存组件？keep- alive

    • 缓存组件

    • 频繁切换，不需要重复渲染，eg：tag切换

    • Vue常见性能优化

    • 与v-show区别：v-show通过css 的display控制，keep-alive通过Vue框架层级控制

  • Vue-router使用

    • 路由模式（hash、H5 history）

      • hash模式（默认）如：http://abc.com/#/user/10

        • 特点：通过url的hash值变化触发路由的变化，实现页面根据hash值渲染对应内容，实现路由跳转
          • hash变化会触发网页跳转，即浏览器的前进、后退 hash 变化不会刷新页面，SPA必须的特点 hash永远不会提交到server端（完全属于前端控制） 实现路由跳转依赖于API window.onhashchange

      • H5 history 模式： 如 http://abc.com/user/20

        • 通过URL变化触发路由的变化，实现路由跳转，并不刷新页面，需要后端支持，通过url无法区分是前端路由还是后端路由。用url规范的路由，但只跳转不刷新页面 需要后端支持 实现路由跳转依赖于两个重要API， history.pushState & window.onpopstate

      • 表现一样，url发生变化，进行路由跳转，但都不刷新页面。

        实现方式不同，hash不依赖于后端，h5 history依赖于后端。

        那么如何抉择使用哪种模式呢？

        to B的系统推荐使用hash，简单易用，对url规范不敏感 to C的系统，可以考虑选择H5 history，但需要后端服务支持

      - 动态路由：

    - 路由懒加载 当打包构建应用时，JavaScript 包会变得非常大，影响页面加载。如果我们能把不同路由对应的组件分割成不同的代码块，然后当路由被访问的时候才加载对应组件，这样就会更加高效。
